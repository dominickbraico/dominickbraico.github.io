Algorithm Theory

üìê Extended Kalman Filter Fundamentals

The Extended Kalman Filter is a recursive state estimator that extends the standard Kalman Filter to handle nonlinear systems. It operates in two phases:

Prediction Phase

  1. State Prediction: Propagate state estimate using system dynamics (IMU measurements)
  2. Covariance Prediction: Propagate uncertainty forward in time

Update Phase

  1. Kalman Gain: Calculate optimal weighting between prediction and measurement
  2. State Update: Correct prediction using GPS measurements
  3. Covariance Update: Update uncertainty estimate

üîÑ Sensor Fusion Strategy

GPS + INS Complementary Characteristics

  • GPS: Provides absolute position with no drift, but low update rate (1-10 Hz) and susceptible to signal loss
  • IMU: High update rate (100+ Hz), continuous measurements, but accumulates drift over time
  • Fusion Benefit: Combines GPS accuracy with IMU continuity for optimal state estimation

üìä State Vector

The system estimates a comprehensive state vector including:

  • Position: x, y, z coordinates (3 states)
  • Velocity: vx, vy, vz velocities (3 states)
  • Orientation: Roll, pitch, yaw angles (3 states)
  • IMU Biases: Accelerometer and gyroscope biases (6 states)

‚öôÔ∏è Key Components Implemented

1. State Propagation

Implemented nonlinear system dynamics using IMU measurements to predict state evolution. Includes proper handling of rotation matrices and quaternion kinematics for attitude propagation.

2. Jacobian Computation

Calculated analytical Jacobians for both the process model and measurement model, required for linearizing the nonlinear dynamics around the current state estimate.

3. Measurement Update

Implemented GPS measurement model and innovation calculation. Includes outlier rejection and adaptive measurement covariance based on GPS quality metrics.

4. Covariance Management

Ensured numerical stability through symmetric covariance enforcement and positive definiteness checks. Implemented Joseph form covariance update for improved numerical properties.

üîß Tuning Parameters

  • Process Noise: Characterized IMU noise through Allan variance analysis
  • Measurement Noise: Tuned based on GPS accuracy specifications and DOP values
  • Initial Covariance: Set based on initialization uncertainty
  • Bias Random Walk: Modeled sensor bias evolution over time

Technical Challenges & Solutions

Challenge: Nonlinear Dynamics

Problem: INS dynamics are highly nonlinear, particularly in attitude propagation.
Solution: Implemented proper quaternion kinematics and computed analytical Jacobians for accurate linearization around current state estimate.

Challenge: Numerical Stability

Problem: Covariance matrices can become non-positive-definite due to numerical errors.
Solution: Enforced symmetry, used Joseph form updates, and implemented periodic covariance reconditioning to maintain numerical stability.

Challenge: GPS Outages

Problem: Filter must maintain accuracy when GPS is unavailable.
Solution: Properly characterized IMU noise and bias evolution, allowing accurate dead-reckoning during GPS gaps while tracking increasing uncertainty.

Challenge: Parameter Tuning

Problem: Filter performance highly dependent on noise parameter selection.
Solution: Used Allan variance analysis for IMU characterization and adaptive measurement covariance based on GPS quality indicators (HDOP, satellite count).
